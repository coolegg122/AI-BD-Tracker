# 双工Debug (Duplex Debug) 工作流程

## 概述
双工Debug系统使用Qwen3和GLM5两个AI agents进行全面的代码审查，通过并行分析、实时讨论和共识达成来确保代码质量。

## 快速开始

```bash
python3 duplex_debug.py <files>
```

工作流程：
1. Qwen3和GLM5并行分析代码
2. 双方实时讨论发现的问题
3. 互相验证对方的发现
4. 生成共识报告和解决方案

## 工作流程详解

### 阶段1: Qwen3初步审查

```
触发: 新代码编写或修改完成
Agent: Qwen3-Debugger
耗时: ~5-10分钟
```

**Qwen3任务:**
1. 分析所有修改/创建的文件
2. 检查审查清单（见qwen3-debugger.md）
3. 识别逻辑错误、安全问题、性能问题
4. 生成初步调试报告
5. 为所有问题分配严重程度

**输出:** `Qwen3调试报告`

---

### 阶段2: GLM5交叉验证

```
触发: Qwen3完成初步审查
Agent: GLM5-Debugger
耗时: ~5-10分钟
```

**GLM5任务:**
1. 独立审查相同的代码
2. 验证Qwen3的发现
3. 寻找Qwen3可能遗漏的问题
4. 关注边缘情况和集成点
5. 挑战任何看似不正确的发现

**输出:** `GLM5交叉验证报告`

---

### 阶段3: 双工讨论

```
触发: 两个agents完成审查
参与者: Qwen3-Debugger + GLM5-Debugger
耗时: ~5分钟
```

**讨论主题:**
1. **共识建立**: 对关键问题达成一致
2. **冲突解决**: 解决任何分歧
3. **优先级对齐**: 确保严重程度评级一致
4. **全面覆盖**: 验证所有领域都已覆盖

**讨论格式:**
```
Qwen3: [提出发现和理由]
GLM5: [验证/挑战并补充观点]
Qwen3: [回应挑战]
GLM5: [提供最终评估]
双方: [对每个问题达成共识]
```

---

### 阶段4: 最终报告

```
触发: Agents达成共识
输出: 综合最终报告
```

**最终报告结构:**


## 使用说明

### 开发者使用
完成新代码后，运行：
```
python3 duplex_debug.py [文件或目录]
```

### AI Agents调用
当被调用进行调试审查时：

1. **Qwen3 Agent:**
   - 读取代码文件
   - 应用审查清单
   - 生成调试报告
   - 传递给GLM5进行交叉验证

2. **GLM5 Agent:**
   - 读取相同的代码文件
   - 读取Qwen3的报告
   - 进行独立分析
   - 生成交叉验证报告
   - 与Qwen3发起讨论

3. **双方Agents:**
   - 讨论发现
   - 解决冲突
   - 生成最终共识报告

## 配置

### 环境变量
```bash
DEBUG_AGENTS_ENABLED=true
QWEN3_MODEL=qwen3-plus
GLM5_MODEL=glm-5
REVIEW_TIMEOUT=600  # 秒
```

### 审查范围
- `backend/` - Python FastAPI后端
- `ai-bd-tracker/src/` - React前端
- `*.py` - Python文件
- `*.jsx` - React组件
- `*.js` - JavaScript模块

### 严重程度定义
- **Critical**: 安全漏洞、数据丢失、认证绕过
- **High**: 功能损坏、重大性能问题
- **Medium**: 用户体验差、代码质量问题、小bug
- **Low**: 样式问题、优化机会、文档

## 维护

### 更新Agent配置
当发现新模式或问题时：
1. 添加到相应的agent配置文件
2. 更新检查清单
3. 在经验教训中记录

### 评估工作流效果
定期评估：
- 问题是否在生产前被发现？
- 是否有误报？
- 是否遗漏了关键问题？
- 工作流是否高效？
